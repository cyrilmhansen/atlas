# DEC-071 - Close Phase 2 mixed and activate Explorer-first Phase 3

## Status

Accepted on 2026-07-15 (`explorer-C`).

## Context

K-M7 concludes Phase 2 with a mixed, informative verdict. Atlas preserves
foreign knowledge, provenance and evidence discipline, and its current registry
helps an agent identify a component. Public qualified selection and generic
composition remain unsupported, while the foreign corpus still lacks competing
implementations.

K-M7 recommended a comparative Knowledge phase before schema 0.2. The human
owner instead selects the documented Explorer-first alternative: make the
existing knowledge and bounded executions substantially more useful and visible
before returning to ontology expansion.

The current static artifact already projects 31 problems, 36 algorithms and 40
implementations and executes five sequence algorithms through the private
visual machine. Its principal product limitation is a flat catalog with weak
entity navigation and comparison, not missing registry data.

## Decision

- Close **Phase 2 - External corpus trial** with verdict **mixed and
  informative** at `docs/phase2/k-m7-phase-audit.md`.
- Activate **Phase 3 - Explorer product trial** with Atlas Explorer active,
  Atlas Knowledge maintained and Atlas Execution Lab frozen.
- Keep schema 0.1, the aggregate YAML registry and Git authority unchanged.
- Use the existing static, local, no-network artifact and five bounded visual
  programs as the initial product baseline.
- Make entity navigation, provenance inspection and factual comparison the
  first vertical slice before adding executable algorithm coverage.
- Keep the Web projection, presentation description, bytecode and WASM facade
  private unless a later decision stabilizes them.

This decision does not authorize schema 0.2, a new general planner, a new visual
instruction, MIR in the browser, a network service, external deployment or a
stable public Web protocol.

## Consequences

- Phase 2's negative selection/composition findings remain visible and are not
  reclassified as product backlog completed by presentation work.
- Explorer may project additional existing registry claims and relations through
  private derived data when a concrete interface workflow requires them.
- Knowledge work is limited to correctness, provenance and compatibility
  maintenance during Phase 3; corpus/ontology expansion is not active.
- Execution Lab changes require a demonstrated Explorer workflow that cannot be
  served by the five accepted programs and a separate scope decision.
- Publication remains explicit. A reproducible local bundle is sufficient for
  initial Phase 3 milestones.

## Alternatives considered

- Comparative foreign selection (`phase3-A`): K-M7 recommendation, rejected for
  the next phase in favor of making existing assets useful to people now.
- Direct schema 0.2 (`schema2-B`): rejected as premature and costly given K-M5's
  private infrastructure ratio and the lack of foreign candidate competition.
- Explorer-first (`explorer-C`): accepted as a reversible shift of active work;
  it preserves all Knowledge evidence and does not preclude a later comparative
  phase.
